API - Описание подключения к рекламной сети plus1 wapstart
=============


1. Вступление
-------------

В документе описано API для подключения к Plus1 как мобильных сайтов, так и приложений.
Задача партнера, заинтересованного в размещение рекламы Plus1 у себя на площадке – своими
средствами согласно данной спецификации сформировать http GET запрос. В ответ на запрос он
получит баннер в одном из доступных форматов, либо техническую заглушку.
Примечание: вы так же можете воспользоваться уже готовым набором инструментов для
подключения к Plus1. Это PHP/Perl исполняемые коды, клиентский Java Script, а так же SDK для
iOs/Android/Java. Данные инструменты доступны в панели управления вашего аккаунта по адресу
plus1.wapstart.ru.

2. URL для запроса
------------------

Запрос к серверу выдачи объявлений идет через хост:
http://ro.plus1.wapstart.ru

*Мы настоятельно советуем использовать именно имя, не IP.

3. Форматы рекламных объявлений
-------------------------------

Наша рекламная сеть позволяет размещать текстовые и графические баннеры, а так же их
комбинации. При этом мы придерживаемся стандартов MMA(pdf).
В текстовом объявлении всегда присутствует кликабельный заголовок до 35 символов, а так же
может присутствовать дополнительный поясняющий текст до 35 символов. Графический баннер
доступен в формате GIF (в т.ч. анимированный), JPG, PNG.


4. Контроллеры и параметры рекламного запроса
----------------------------------------------

Контроллеры, обрабатывающие запрос, делятся на две группы:
- приложение (application)
- сайт или сервер рекламной сети (viewBanner и viewBannerXml).
Параметры можно передать двумя способами:
- в query_string запроса
- в дополнительных заголовках запроса.
В зависимости от выбора того или иного контроллера набор передаваемых данных будет отличаться.

5. Установка рекламы в приложения
----------------------------------

При обращении к хосту API из приложений контроллер выглядит следующим образом:
/?area=application&version=2
*где version равный 2 символизирует использование соответствующей версии API

В query_string необходимо передать следующие параметры:

* *id* - идентификатор площадки в сети Plus1. Беззнаковое целое. Обязательный параметр.

* *pageId* - идентификатор показа на странице, строка [0-9a-f]{40} В рамках одного pageId ротатор будет пытаться найти разные баннеры. Для показа нескольких баннеров на одном экране вы должны
сделать запросы с одинаковыми pageId. Обязательный параметр.

* *encoding* - идентификатор кодировки. 1 для utf-8 и 2 для cp1251. Не обязательный параметр. По-умолчанию 1.

* *sex* - пол пользователя. 1 для мужчин и 2 для женщин. Необязательный параметр.

* *age* - возраст пользователя. Число полных лет. Необязательный параметр.

* *kws* – Ключевые слова пользователя. URL encoded строка, содержащая значения, разделенные запятыми. Необязательный параметр.

* *login* - логин, либо другой идентификатор пользователя. Строка. В целях сохранения конфиденциальности личной информации мы рекомендуем передавать не логин, а результат
хеширующей функции от него. Необязательный параметр.

* *location* - географические координаты пользователя в формате WGS-84. В параметре через точку с запятой указываются долгота и широта (в градусах), например, location=37.677751;55.757718
Необязательный параметр. 

Все параметры, характеризующие устройство (бренд/модель/операционную систему), должны быть
переданы дополнительными заголовками.

Из них обязательными является:
* *User-Agent* - подчиняется стандартным правилам
* *Cookies* - требуется передавать cookie, которую ротатор пытался поставить при первом запросе
* *Host: ro.plus1.wapstart.ru* - этого требует http/1.1

Желательно так же передавать следующие заголовки:
* *x-display-metrics* - разрешение экрана в формате "1024x768".
* *x-application-type* - в общем случае название платформы. Строка с произвольным содержимым.
* *x-device-imei* - imei номер устройства. Целое число.
Дополнительно можно передать:
* *x-preferred-locale* - язык пользователя. Строка.

Так же можно передать любые другие заголовки, которые позволят более точно таргетировать
пользователя, предварительно согласовав их с администрацией Plus1.

5.1 Порядок разбора ответа
--------------------------

Приведенная блок схема кратко описывает порядок разбора ответа от API:
Пример запроса к контроллеру application будет выглядеть следующим образом:


    GET /?area=application&version=2&id=8&sex=1&age=42&kws= russia%2Cmobile%20adv%2Cwapstart&pageId=0000000000111111111122222222224444444444&location=42;43 HTTP/1.1
    Host: ro.plus1.wapstart.ru
    User-agent: Android, linux, htc hero
    Cookies: wssid=e1f9c9ebab3014bc65940ab812c8629bf25dd86e
    x-display-metrics: 480x320
    \r\n\r\n

И пример ответа:

      <responseType>1</responseType>

      <pictureUrl></pictureUrl>
      <pictureUrlPng></pictureUrlPng>

В данном случае наш сервер нашел баннер, подходящий под этот запрос и вернул в ответ xml, содержащую следующие поля:

* *id* - идентификатор баннера в plus1 (справочно)
* *title* - заголовок баннера
* *content* - текст баннера
* *responseType* - тип обратной связи. На момент написания документа используется только 1, что означает "переход по ссылке"
* *link* - ссылка, которую нужно открыть в стандартном браузере при клике пользователя по баннеру
* *pictureUrl* - url с картинкой баннера
* *pictureUrlPng* - url с картинкой баннера (копией) в формате png

6. Формирование запроса сервером (web сайта или рекламной сети)
--------------------------------------------------------------------------------------------------

В текущей реализации API используется два контроллера:
* viewBanner - служит для отдачи html блока с баннером
* viewBannerXml - для отдачи баннеров в формате xml.

При обращении к хосту API контроллер выглядит следующим образом :
/?area=viewBanner&version=2 или /?area=viewBannerXml&version=2
*где version равный 2 символизирует использование соответствующей версии API

Примечание: в качестве примера реализации подобного запроса рекомендуется ознакомиться с классом Plus1BannerAsker.class.php, который реализует описанную ниже логику на языке php.

В query_string необходимо передать следующие параметры:
* *id* - идентификатор площадки в сети Plus1. Беззнаковое целое. Обязательный параметр.

* *pageId* - идентификатор показа на странице, строка [0-9a-f]{40} В рамках одного pageId ротатор будет пытаться найти разные баннеры. Для показа нескольких баннеров на одном экране вы должны сделать запросы с одинаковыми pageId. Обязательный параметр.

* ip* - ip4 адрес клиента. Обязательный параметр.

* *encoding* - идентификатор кодировки. 1 для utf-8 и 2 для cp1251. Не обязательный параметр. По-умолчанию 1.

* *sex* - пол пользователя. 1 для мужчин и 2 для женщин. Необязательный параметр.

* *age* - возраст пользователя. Число полных лет. Необязательный параметр.

* *kws* – Ключевые слова пользователя. URL encoded строка, содержащая значения, разделенные запятыми. Необязательный параметр.

* *login* - логин, либо другой идентификатор пользователя. Строка. В целях сохранения конфиденциальности личной информации мы рекомендуем передавать не логин, а результат хеширующей функции от него. Необязательный параметр.

* *geoData* - строка с гео-информацией (город, штат, страна разделенные знаком “;”). Если эта информация доступна, пожалуйста уведомите администрацию сети для окончательного согласования
формата. Необязательный параметр.

* *countryCode* – Код страны конечного пользователя в формате ISO-3166-1 (2 латинские буквы). Необязательный параметр.

* *markup* - тип разметки (применим только для viewBanner) 3 для xhtml и 2 для wml. По умолчанию 3. Необязательный параметр.

* *bannerAmount* - число баннеров. Беззнаковое целое от 1 до 3. Необязательный параметр. По умолчанию 1.

Параметры, которые надо передать заголовками:
Все заголовки пользователя, начинающиеся с "accept-" и "x-" а так же "host", "referer", "via",
требуется передать вместе с запросом в API, добавив перед именем заголовка подстроку "x-plus1-".

Дополнительно нужно передать в ротатор заголовок "x-plus1-remote-addr", поместив в его
значение ip адреса клиента (remote-addr).

Значение заголовка User-Agent требуется продублировать в этом же заголовке без изменений
(т.е. передать и x-plus1-user-agent и user-agent).

Обеспечение определения уникальности пользователя:
По возможности старайтесь обеспечивать API сведениями об уникальности пользователей. Для этого можно использовать два механизма
* передача параметра login
* передача в ротатор Cookie с именем wssid. При этом простановку куки конечному клиенту вы
должны обеспечить своими силами.

В xml баннере (контроллер viewBannerXml) есть нода с именем cookieSetterUrl. В ее содержимом находится картинка, которую надо "обязать" загрузить пользователя. Как правило, для этого надо
просто вывести ее в html.
В обычном (html) баннере тэг с этой картинкой присутствует по-умолчанию.
